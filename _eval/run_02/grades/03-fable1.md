**Best thing:** Choosing a strict whole-document XML parser (roxmltree) and funneling every failure mode through one explicit error ladder — the combination made "malformed XML is a recorded fetch failure" true by construction, and it is the entire difference between this repo's 26/26 and the rest of the field's shared C9/H12 failure. The e2e then pinned that exact behavior against feedgen's own truncated fixture.

**Worst thing:** feedctl's exit-code contract (0/1/2, spec-pinned) is documented but has zero test coverage — and `refresh_all`'s `unwrap_or_default()` (refresh.rs:122) silently turns a DB failure into "no feeds refreshed," returning 200 with an empty array where a 500 belongs.

**Narrative (≤150 words):** The only 26/26 in the field, and the margin is architectural rather than lucky: where six repos hand-drove a lenient streaming parser and inherited quick-xml's silence about truncated documents, this one parses the whole document strictly and routes every failure — network, parse, even the database apply — through a single recorded-error ladder. The same judgment shows elsewhere: dual text+integer date columns, offset-preserving URL decoding mirrored across client and server, an e2e that asserts the exact malformed-feed contract over real HTTP. The suite around the edges is mid-field, though: exit codes, pagination, search, and tie-breaks ride on manual verification, and one swallowed DB error hides behind a 200. Delivered at 0.92× the control's credits with clean five-commit hygiene and no ceremony. The strongest artifact of run 02 — and the only one whose parser treats malformed input as the spec demands.
