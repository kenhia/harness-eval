**Best thing:** feedgen's request log exists specifically so the test suite can assert what feedd *sends* — turning "store ETag/Last-Modified and send If-None-Match/If-Modified-Since" from a write-only claim into an observed behavior. Together with PLAN.md's three preserved decision reversals, it's the field's best evidence of a verification loop that changed the outcome.

**Worst thing:** The comment at parser.rs:183 — "Malformed documents must surface as an error" — sits directly above the `Event::Eof` break that lets truncated documents through, with the open-element stack (`st.stack`) unchecked right there; 165 tests and five malformed-feed test cases all miss it because the repo's only malformed fixture is the one flavor the parser does catch.

**Narrative (≤150 words):** The most heavily engineered artifact of the field, and — unlike run 1.5 — the extra spend mostly bought things the spec priced: the 6,290 Rust lines are over half tests (165 of them, including the only exit-code matrix and the only proof that conditional-GET headers are actually sent), and PLAN.md is a genuine design record that preserves its own reversed decisions. Craft is impeccable: no dead code, no unwraps, envelope-correct errors even for extractor rejections, dual-column instants with a leap-second clamp. The failure is therefore the more instructive: a stated guarantee ("malformed documents must surface as an error") implemented for the malformation class quick-xml reports, tested against a fixture of exactly that class, and silent for the truncation class the suite probed. Cost lands at 1.3–2× the control depending on unit, inside a measured 44% variance band — modest overhead for the field's best test asset.
